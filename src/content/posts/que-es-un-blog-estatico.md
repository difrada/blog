## La web no siempre fue complicada

Cuando naces en la era de las apps, todo parece requerir servidores, bases de datos y APIs. Pero la web empezó siendo archivos de texto que un navegador leía directamente. Eso es, esencialmente, lo que es un sitio estático.

## ¿Estático vs dinámico?

Un sitio **dinámico** genera cada página en el momento. Cuando visitas una página, un servidor ejecuta código, consulta una base de datos, arma el HTML y te lo envía. Esto toma tiempo y recursos.

Un sitio **estático** ya tiene todo pre-generado. Son archivos HTML listos para servir. No hay base de datos, no hay código ejecutándose por cada visita.

## ¿Por qué elegimos esto?

Tres razones simples: velocidad, seguridad y simplicidad. Nuestro blog carga en milisegundos, no tiene superficie de ataque (no hay servidor que hackear), y el contenido vive en archivos de texto que cualquiera puede editar.
